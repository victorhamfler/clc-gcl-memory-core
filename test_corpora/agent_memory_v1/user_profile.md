# User Profile: Mira Vale

Mira Vale is the synthetic user for the memory experiment. Mira is building local AI tools and wants an assistant that remembers project-specific constraints across sessions.

Mira's working preferences:
- Mira prefers Windows PowerShell commands for commits and repository work.
- Mira uses WSL for model files and Linux-based ML runtimes, but does not want Git commits made from WSL.
- Mira wants local experiments before publishing changes.
- Mira prefers practical summaries over long theoretical explanations.
- Mira wants the assistant to say when a database, server, or repository was changed.
- Mira wants the assistant to avoid uploading to GitHub unless explicitly instructed.

Important user facts:
- The user's main development folder is a Windows project folder.
- The user values reversible experiments and temporary databases.
- The user is comfortable iterating with test corpora, feedback labels, and memory ranking experiments.
- The user wants the agent to improve through explicit feedback rather than hidden fine-tuning.
